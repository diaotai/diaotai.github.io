精通CSS笔记

Posted on 2019-09-21

语义化 CSS

  • 给元素添加类名时，即使类名明确用于样式，也不要以体现其视觉效果，正确的作法是用类型指出该组件的类型。比如说，将商品列表命名为 product-list 而不是 larged-centered-list
  • 在使用选择器时，不应当涉及太多的上下文
  • 要从结构中分离出皮肤，从内容中分离出容器，不要将字体、颜色和定位、浮动的样式写在一起
  • 要注意以下几点:
    • 以类名作为应用样式的主要手段
    • 类名要能顾名思义，清晰明了
    • 通过拆分出单一用途的规则来避免不必要的重复
    • 不要把元素类型和样式规则绑定到一起

CSS 选择器

子代和同辈选择符

  • a > b: 所有子代
  • a + b: 所有相邻同辈属性（位于 a 之后相邻的 b）
  • a ~ b: 所有同辈组合子，用于选择位于 a 之后的所有 b

属性选择器

  • a[title]: 拥有 title 属性的 a 标签
  • a[title=”submit”]: 拥有 title 属性为 submit 的 a 标签
  • a[href^=”abc”]: title 属性以 abc 开头
  • a[href$=”abc”]: title 属性以 abc 结尾
  • a[href*=”abc”]: title 属性中包含 abc 三个字符
  • a[href~=abc]: 经空格分隔开的值中有一个为 abc

伪类

  • target: 它匹配的元素拥有 id 属性，且为当前页面 URL 末尾的 # 后边的值
  • not(): 反选伪类
  • nth-child(3n + 4): 该元素的 第(n=0,1,2) 个元素
  • nth-child(-n + 4): 当 n 为负值时，即为前四个(nth-last-child 即为逆序)
  • only-of-type: 会选择特定类型的元素，如果为纯数字，则为总体而言的第 n 个（必然只出现一次）

  • 标注为 !important 的开发者样式

  • 用户样式
  • 浏览器默认样式

字体

  • em 对字体是百分比叠加的（如果是用在 font-size 上是以父元素的 font-zize 为基准）
  • 行高指的是行盒子的总高度，更通俗的是行间距，排版属于叫铅空，就是排字用来隔开字符的铅块。css 中的铅空会始终同时应用于行盒子的上方和下方。
  • line-height 设置数字意味着以当前 font-size 为基准，子元素继承的会是一个系数，始终以自身 font-size 为基准
  • vertical-align 的默认值是 baseline，即子元素的基线与父元素的对齐
  • font-variant 可以通过 small-caps 把英文文本转换成所谓的小型大写字母，即全部字母都大写，但是只有第一个是正常大小，其它的字母的大小会和小写时一样
  • 使用新 css 属性 hyphens 可以在连续英文中插入连字符
  • column 模块可以将网页中部分的内容转换成类似与报纸的分栏效果，而 colimn-span 可以营造出横跨多栏的效果
  • 在排版时常见的是以正文行高的整数倍来进行设计，因此，margin line-height 的和应该也是整数倍
  • 潜入 web 字体的关键是 @font-face 规则，其中必须要有的是 font-family 和 src 属性，而 font-weight 和 font-style 则是可选的，后两者不会改变 web 字体，但是会影响匹配的优先级——浏览器会优先匹配正确的字体，正确的 font-family 会优先于正确的粗细值。
  • web Font Loader 可以报纸跨浏览器的字体加载的一致性（统一 FOCUS），匹配后备字体可以进行微调来减少抖动
  • 文字阴影通常用于标题或短文吧，需要指定 x，y 的偏移量以及模糊距离和颜色值，多组阴影会进行这边，先定义的在下，后定义的在上。

可见格式化模型

盒子模型

  • 组成盒子模型的范围有 width, height, padding, border, margin, outline 类似于 border，但是并不影响盒子和宽度和高度，因此可以用来复杂布局的调试。
  • 默认情况下，width 和 height 都指的是内容区的宽度和高度，但这样其实蛮坑爹的
  • 通过将 box-sizing 的值修改为 border-box，那么 width 和 height 属性的值将包含内边距和边框，不过 margin 不会算在其中
  • 在 CSS 中设置任何高度值都应该慎重，这是因为元素的高度通常应该取决于所包含的内容，一个定值容易产生越界。如果需要使用，也最好使用 min-height。

可见盒子模型

  • 行内盒子是沿文本流水平排列的，也会随着文本换行而换行。它们之间的水平间距可以通过水平方向的 margin、padding、border 来调节。而在竖直方向，唯一修改盒子大小的方法只有 line-height（line-height代表最小值）。
  • 当文本没有被任何元素包裹的时候，即为匿名盒子，所有展示都位于盒子中
  • 外边距折叠：外边距折叠只发生在文档常规文本流的垂直方向上，行内盒子、浮动盒子和绝对定位盒子的外边距不会折叠。当发生外边距折叠时，最终结果取较大的那个值，当出嵌套时，如果没有 border 或 padding 或者某些具体的内容在进行隔离，同样会出现外边距折叠，一个空的块级元素自身的上下边距也会折叠。在外边距折叠后，如果碰到其它外边距，还会继续折叠。
  • 包含块是计算内边距和外边距百分比的依据，在 static 中，默认使用父元素作为包含块。

浮动

  • 浮动元素会自动收缩为适应其中内容的最小宽度，除非显式设置的 width
  • 如果浮动元素太窄，无法容纳所有浮动元素水平排列，那么就会向下移动。
  • 由于浮动元素脱离正常的文档流中，因此无法在文档流内部生成高度，因此需要在结束的的 div 标签签名额外添加一个空元素，然后清除该元素。此外，还可以通过 ::after 伪类生成 content 来处理问题。

    格式化上下文（BFC）

  • 页面必须包含突出的浮动元素，而且所有块级盒子左边界都默认与包含块左边界对齐。这组规则即为块级格式化上下文。外边距折叠只会发生在属于同一块级格式化上下文的块级元素中。
  • 以下方法为生成块级格式化上下文的方法：
    • display 为 inline-block 或 table-ceil之类的元素
    • float 属性值不为 none 的元素
    • 绝对定位的元素
    • overflow 不为 visible 的元素
  • 浮动元素从文档流中脱离后，会触发其后的元素中行盒子的缩短行为，而当一个元素具备了触发新跨级格式化上下文的条件，且挨着浮动元素后，就会收缩至适当大小且保证做外边必为直线。

水平布局

层叠

  • z-index 并不是在所有层面都生效的，只有在非 static 或几种特殊情况下（opacity < 1,transform!=none,filter!=none），才会有效果。要注意，z-index 只有在父级层叠上下文中才有意义，子级层叠上下文会被视为单独的一部分。

    水平布局

  • 要让两个元素的内容竖直方向对齐，先把它们设置为 inline 属性，然后应用 vertical-align: center
  • 在 HTML 的行内，可能出现将换行视为一个空格的情况，这组情况下需要将 font-size 设置为 0 来解决
  • 通常只将行内块当作兼容旧版本浏览器的逻辑，正常而言应当使用更为先进的 flexbox
  • 表格布局与常规块不同，不设置宽度的表格会“收缩适应”内容宽度，除非包含块中的内容将其撑开，但是表格布局有很多弊端，比如无法应用外边距，再比如给其定位时可能产生一些无法预料的问题

    Flexbox

  • 在 flex 中，若不指定大小，那么 flex 容器内的项目会自动收缩，也就是说，会变成容纳各项的最小宽度（如果主轴为列那就是高度），以恰好能容纳自身内容为限。
  • 如果某一侧外边距值为 auto，且在容器那一侧还有空间，那么该外边距就会扩展至占据所有位置。
  • align-items 的默认值为 stretch，也就是说，会拉伸至充满父容器
  • 其实 flex + margin:auto 就可以完成单个元素的居中
  • flex-basis 的默认值为 auto，即为取默认的宽或高，如果没有设置，则收缩为可容纳的最小宽度，当设置为 0 时，flex-grow 将不再考虑其宽度所带来的影响
  • flex-grow 的默认值为 0，即默认不再扩展
  • flex-shrink 即为收缩，考虑时将使用 flex-basis * flex-shrink 作为比例系数来进行缩放
  • flexbox 的方向是逻辑方向，即以文本方向决定哪里是开头哪里是结尾
  • order 可以在不改变 html 的情况下改变展示顺序，order 越小越靠前，相等则按照原顺序
  • 为了处理 flex-box 换行后过于庞大的情况，可以给 item 设置最大宽度
  • 与常规块不同，不需要将可伸缩项设置为非 static 的定位值，也可以对它们应用 z-index 属性，并产生 z-index 上下文
  • flex 中可伸缩项具有隐性最小宽度，即收缩不可能收缩到无法展示内容的情况下，为此，可以覆盖 min-width，明确一个主尺寸

盒子模型

  • 颜色的 hsl 语法需要传入三个参数，第一个是角度，第二个和第三个分别代表饱和度和亮度
  • opacity 设置之后，是无法再让其子元素变得不那么透明的

    背景图片

  • 在使用背景图片设置位置时，设置具体像素值代表着包含块左上角具体图片左上角的距离，而百分比则代表着包含块左上角到图片对应百分比点的距离正好为包含块的百分比。
  • 背景图片默认展现在 border 范围内，而 background-clip 属性可以改变这个行为，这个属性的默认值为 border-box，而 padding-box 和 content-box 可以解决这个问题。
  • background-attachment 可以解决背景图片附着的问题
  • background-size 是百分比并不是根据图片固有大小，而是根据包含块的大小来测定的
  • background-size 的 contain 和 cover 都不改变 宽高比例，前者可以保证完全被展示，后者会导致被剪切

    border

  • border 的顺序是左上角开始顺时针旋转
  • border 的值指的是圆的半径，若两个圆角的弧线相交，则两个轴向都会分别缩小半径，知道圆弧不再相交。对于方形元素的对称圆角而言，任何大于 50% 的值都会得到圆形。而对于矩形元素，则可以设置一个非常大的值来实现胶囊效果。
  • 当矩形变为圆角后，元素对布局的影响和之前是一样的，但是响应点击的范围是以变化之后为标准的。

    盒阴影

  • 头两个值表示 x 轴和 y 轴的偏移，第三个值表述模糊半径，最后一个值是颜色。
  • box-shadow 可以使用 inset 来设置内阴影，从而作出一种被镂空的效果

    渐变

  • 渐变必然是从一个方向到齐对称方向，默认是从上到下，坐标轴垂直向上，方向为顺时针。

    嵌入图片的改变

  • max-width 100% 是限制图片的方法，这意味着会随着容器的缩小而变小，但不会大到超过自身尺寸，且不会改变其比例。
  • 通常会加上 width、height 为 auto 的声明
  • object-fit 可以像 background-size 一样来设置图片的大小，默认是 fill，会拉伸图片来充满容器，因此可能破坏宽高比。none 代表使用图片自有宽高，cover 和 contain 与 background-size 相同，scale-down 则会从 none 和 contain 中选一个，以小的为准
  • 没有设置宽高的可替代元素最终的默认大小可能是 300px 或 150px
  • 内外边距的百分比是以本身的宽度来确定的，因此可以设置为宽度 100%，padding-bottom 75% 来完成视频的包含

页面布局与网格

  • 传统行列布局的方法是通过 row 和 col 的百分比来计算宽度，通过 padding 来完成边距自适应
  • flex 整体而言是更好的旋转，但是 flex box 会随着其中内容的加载而重新计算尺寸，因此在页面首次打开的时候会调一下，体验并不好

    二维网格布局

  • 分别通过 grid-template-rows 和 grid-template-columns 来设置宽高，1 fr 代表可分配宽度分之一，这个会放到最后来计算
  • minmax 可以设置以下最小值
  • 当 grid-row 或 grid-column 设置跨所有行时，可以将最后一个置为 -1
  • justify-content 和 align-content 可以设置多个个体的排布
  • 而 align-self 可以设置单个在副轴上的排布
  • grid-auto-flow 可以设置默认主轴的方向
  • 默认排布模式是会导致空洞的（出现较大的元素），如果设置为稠密模式（grid-auto-flow: row dense）会自动跑多变，每次都从头开始，尽力找到最前面得到空位置
  • 可以通过 grid-area 来给网格空间命名

响应式 web 设计与 css

  • 理想视口即手机应该有的大小，而可见时刻为实际显式的内容，布局视口则是放大后的整个页面的内容
  • 设置 meta 标签可以很好的使用理想尺寸，此外，css理论上也可以做到，在 style 标签中使用 @viewport 即可，最好放在 html 里，以保证被尽早执行
  • 响应式 web 的核心在于媒体查询，媒体查询可以使用一连串的逗号来解决，逗号相当于 或
  • 媒体查询中使用的 em 始终相对于浏览器偏好中的基准字段，而不是可以通过 css 调整的 html 元素的 rem
  • 响应式设计应当优先于移动端，flexbox 和 grid 其实都是很不错的旋转
  • 通过 img 的 srcset 属性，可以让浏览器对图片自有进行预处理，从而让图片等资源在完成整个页面晕眩之前就进行加载
  • srcset 可以针对不同的大小和分辨度来使用不同的图片
  • sizes 属性可以针对当前的图片来决定图片的大小
  • 可以在 picture 属性中使用 webp，其中有 source 和 img，img 可以作为整体的后备

变换、过度与动画

变换

  • transform 不会影响布局本身，但是所有点都会被移到新位置上，图形所在的整个局部坐标系统都会被旋转，同样，旋转也不会影响页面其它部分的布局
  • 变换默认是以元素边框盒子的中心为原点的，可以通过 transform-origin 来设置 1 到 3 个值，如果忽略，则为 center。坐标原点在图形左上方。
  • svg 的变换有一些不同，比如 transform-origin 默认即为左上角，而非中心。
  • translate 的百分比是相较于自身而非包含块
  • 多重变换的顺序非常重要。如果先旋转伪元素，那么变换会以旋转后的坐标来完成
  • 应用给元素的任何变换都会创建一个堆叠上下文，这意味着在使用变换时，要注意 z-index，因为变换后的元素有自己的堆叠上下文，也就是说，子元素上的 z-index值再大，也无法和父堆叠上下文呢相比较。
  • 单写某个 transition 形式无法制定多个属性，但可以指定多个不同的过度，以逗号进行分隔。
  • transition-property 指定的值决定了要应用的过度数量，如果其它过度列表持续时间更短，则会重复。
  • steps 可以将动画分为不同的几帧。
  • transition-delay 可以用来延迟动画的触发，延迟时间可以为负值，这样可以跳过一部分动画。
  • 关键帧动画中 css 属性不允许添加 ！important，加了会被忽略。
  • 直接修改 dom 的 css 属性可以触发 dom
  • 在进行 x，y 轴的旋转时，会涉及到透视的概念，用户距离透视点越近，影响越大，反之越小，默认无穷远，即不影响。
  • 默认情况下，嘉定观察者的实现与应用透视的元素相较于元素的中心。通过 perspective-origin 属性来修改消失的点的位置。
  • backface-visibility 属性可以用来控制背面展示，默认是 visible，不过可以设置为 hidden
  • 当进行三维反转时，所有子元素的三维反转都会失效，因此需要通过 transform-style 属性来设置一个三维上下文， 改制为 perserve-3d.

高级特效

  • shape-outside 可以应用给浮动元素，不改变自身的外观，而是通过设置其形状影响外部的文档流
  • 当 shape-outside 设置为 border-box 时，此时设置 margin 会呈现 border-box 的形状
  • 剪切会影响对象的相应区域，而蒙版不会
  • 剪切使用 clip-path 来进行，只影响文档渲染后的外观和相应区域，并不会影响文档流。
  • 剪切可以通过 svg，具体而言是在 svg 里面设置 id，在引用时使用 url + id 的形式来解决。
  • 由于 svg 下剪切路径硬编码的坐标，因此无法缩放，可以将其设置为大小为 1 的 svg 来解决这个问题（换成了对象边界坐标系）
  • 为保证正确的几个，我们要遵循应用剪切、蒙版、混合和滤镜的标准次序

原理

浏览器如何解析 css

  • 解析文件即构建对象模型（将 html 解析为 dom 对象节点树，在浏览器碰到指向 css 的链接时，会获取并解析 css，将 css 文件转化为 css 对象模型，同样是树型结构）
  • 构建渲染树（被隐藏的 DOM 节点不会出现在这里，而类似于 :before 的伪元素则可能出现在其中，但是在 DOM 中不会有，除了节点，浏览器还会在这里保存其它表现性信息，如滚动块或视口）
  • 布局（根据渲染树确定不同对象显式于页面的什么位置）
  • 给予渲染树的信息进行绘制（如果浏览器发现最终图像的某一部分不影响页面其他内容的显式，也可能会拆分成不同的任务，每个任务负责绘制特定的部分）
  • https://csstriggers.com/ 标注了每个属性对应渲染的什么阶段
  • will-change 属性可以用来进行硬件加速，但是一定要慎用

性能

  • 尽量减少 HTTP 请求，将 css 控制在 1～2 个范围内，注意，@import 同样会触发 HTTP 请求
  • 压缩和缓存可以有效的加快加载速度
  • script 会阻塞渲染，而 async 会异步加载脚本，但是会在脚本加载完毕时执行，阻断 HTML 解析。如果时 defer 的话，同样会异步加载脚本，但是会等到 HTML 解析结束后再执行加载的脚本

杂项

  • 如果希望根据浏览器是否支持某个 CSS 属性来提供不同的样式，那么可以在 css 中使用 @supports 模块，该代码被成为条件规则
