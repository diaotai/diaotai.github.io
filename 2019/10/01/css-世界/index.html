css 世界

Posted on 2019-10-01

基础流布局简介

  • 流实际上是 css 世界中的一种基本定位和布局机制，具备自适应的特性。流对于某些 table 并不适用。
  • 通常把元素分为块级元素和内联元素
  • width 为 auto 时如果是块级元素会充分利用可用空间，如果是浮动、绝对定位、inline-block 时会收缩到合适大小，如果是内容连续的很长的英文或数字，则会超出（word-break:break-all 可以用来解决该问题）
  • 如果内部没有块级元素或块级元素没有设定宽度值，则最大宽度实际上是最大的连续内联盒子的宽度
  • 对普通流里的元素，如果 height:100% 要想生效，其父级必须有一个可以生效的高度值（否则百分比和 auto 无法计算）
  • width 不存在以上问题，它会使用包含块的真实的计算值作为百分比计算的基数
  • 正常布局的宽高百分比是以包含块的 content-box 来计算的，而绝对定位的宽高百分比是相对于 padding-box。
  • max-width 的优先级要高过 width，而 min-width 的优先级要高过 max-width
  • 通过设置 max-height 可以完成高度展开的动画
  • 每个行框的开始都会有宽度为 0 高度为 line-height 或 font-size 的内联块，用于撑起基本的高度(可以通过设置一个 inline-block 的空 span 来体现)

盒尺寸四大家族

content

  • 所有替换元素都是内联元素，其内容不受页面 css 上的影响，有着自身的尺寸
  • 若替换元素的固有尺寸有这个宽高比例，那么设置一个就会自动形成另一个，将图片设置为 block 不会影响其尺寸规则
  • 替换元素和非替换元素只隔了一个 content 属性，但是替换元素的固有尺寸是无法设置的
  • content 只能改变视觉呈现，而当右键保存时，还是原来的内容
  • content 生成的文本无法被选中，此外，还不能匹配 empty 类

    padding

  • 内联元素的竖直方向的 padding 是影响布局的（具体体现在会出现滚动条）
  • 对于非替换元素的内联元素，不仅 padding 不会加入行盒高度的计算，margin 和 border 也是如此，都是不计算高度，但实际上在内联盒周围发生了渲染
  • 内联元素的高度完全是受 font-size 的限制
  • 很多表单元素都内置了 padding

    margin

  • 只有当元素处于充分利用可用空间状态的时候，margin 才可以改变元素水平或竖直的可见尺寸
  • chrome 的滚动条为子元素大小超过 content-box 时出现，而 firfox 和 ie 是超过 padding-box 时出现，因此，子元素只能使用 margin-bottom 留白，而不是 padding-bottom
  • margin 和 padding 的百分比无论水平还是竖直都是按照块度来计算的
  • 在没有设置宽高还会自动填充的情况下，margin 如果一侧为定值，一侧为 auto，则 auto 占据剩余所有空间，若都为 auto，则平分距离
  • 当内部元素尺寸大时，说明可用空间都没有了，会当作 0 来处理
  • 内联替换元素垂直 margin 有效，不会发生 margin 合并的情况

    border

  • border-color 的默认值为 color 的值

内联元素与流

  • vertical-align: middle 并不是绝对的垂直居中对齐，只是一种近似效果
  • ex 可以用来保证与文字大小相一致
  • 行内元素的大小是 line-height 撑开的，line-height 保证了最小的高度，对于非替换元素的纯内联元素可以认为高度完全由 line-height 绝对
  • 内容区域高度受 font-family 和 font-size 双重影响，而 em-box 仅受 font-size 影响，通常内容区域高度要更高一些。
  • 内联替换元素和非替换元素放在一起时会共同形成一个行框盒子，line-height 决定其最小高度
  • css 中，计算行高的时候，一定不要向下舍入，而是要向上舍入
  • vertical-align 可以使用值来计算，负值向下，正值向上。
  • vertical-align 只能作用域内联元素和 display 为 table-cell 的元素上
  • 当为 table-cell 时，vertical-align 设置在自己身上，作用域子元素上
  • 把 font-size 和 line-height 一起设置可以解决间隙的问题
  • 非主动触发唯一的内联元素是不可能跑到计算元素之外的

流的破坏与保护

float

  • 浮动参考的元素是浮动元素对齐参考的实体，通常是行框盒子
  • clear 属性是让自身不能与前面的浮动元素相邻，根本不会对之后的浮动元素造成影响
  • 如果 clear:both 元素前面的元素就是 float，那么 margin-top 负值即使设置成 -9999 px 也不会有任何效果。

    BFC

  • display 为 inline-block、table-caption、table-cell
  • position 不为 static 和 relative 时
  • overflow 不为 visible 的块元素
  • contain 值为 layout、content或 paint 的元素
  • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）
  • 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
  • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

    overflow

  • 清除浮动最合适的不是 clear，而是 overflow，设置 为 hidden。如果要完成元素裁剪的同时在四周设置间隙，那么可以使用透明边框，此时内监局为 padding 时无能为力的。
  • 如果两个方向的 overflow 有一个是 visible 而另一个不是，那么 visible 会表现为 auto。
  • 要想实现 text-overflow: ellipsis,必须同时设置另外两个属性– white-space:nowrap,overflow:hidden。
  • 锚点定位（使用 id 进行跳转）会发生在普通的容器上，且由内向外（指如果存在多层会从最里面那层逐层向上）。
  • overflow:hidden 里的元素也是可以滚动的，可以通过锚点来进行跳转。

    absolute

  • 当 absolute 和 float 同时存在的时候，float 属性是无任何效果的。
  • absolute 使用的包含块是 padding-box 而非 content-box
  • 行内元素生成的包含块由前后内联盒子决定，和里面的细节毫无关系（比如说里面字体特别大撑开）
  • absolute 如果不设置 top 和 left 的话会处于原位置，只是会脱离文档流。

    absolute 和 overflow

  • 如果 overflow 不是定位元素，同时绝对定位元素和 overflow 容器之间也没有定位关系，则 overflow 无法对 absolute 元素进行裁剪，而如果 overflow 的属性值不是 hidden 而是 auto 或者是 scroll，即使绝对定位元素高宽比 overflow 元素高宽还要大，也不会出现滚动条。
  • absolute 被裁剪或 fixed 固定定位失效时，可能是 transform 的原因，

    absolute 其它

  • 当 absolute 只设置某个方向时，另一个方向的定位依旧保持相对定位特性。
  • clip 要起作用其元素必须是绝对定位或者固定定位

    relative

  • 相对定位的百分比值是给予包含块计算的
  • 当相对方向的属性值同时出现时，会取上左方向
  • 在使用 relative 时，力求范围较小
  • 使用普通元素替换 fixed 可以解决很多问题

层叠

  • 在同层叠关系下，后来居上，大者为上。

    基本层叠关系（自下而上）

  • 层叠上下文
  • 负 z-index
  • block 块状水平盒子
  • float 浮动盒子
  • inline 水平盒子
  • z-index 为 auto 或 0
  • 正 z-index

    层叠上下文

    层叠上下文的作用

  • 层叠上下文提供了一个基准，如果层叠上下文 A 在层叠上下文 B 之下，那么 A 上的元素绝对不可能位于 B 之上，无论 z-index 是多大
  • 层叠上下文内采取 z-index 谁大谁在上面的措施（其实没什么卵用，正常的文档流根本用不着 z-index，如果要使用必然已经触发的位移）
  • z-index 为负数时，依旧无法突破当前上下文。z-index 负值的渲染就是寻找第一个层叠上下文的过程，然后层叠顺序止步于这个层叠上下文元素。

    如何产生层叠上下文

    z-index 派（对于它们，产生的层叠上下文顺序为 z-index 的值）

  • 当 position 为 relative、absolute 且 z-index 不为 auto(其实 auto 时也产生了，可以认为 z-index 为 0)
  • z-index 不为 auto 的 flex 或 grid 子项（此时 z-index 必须为数值）
  • position 为 fixed 或 sticky

    其它（此时都认为 z-index 为 0）

  • opacity 属性值小于 1 的元素（参考 the specification for opacity），
  • transform 属性值不为 “none”的元素，
  • mix-blend-mode 属性值不为 “normal”的元素，
  • transform属性值不为“none”的元素
  • filter值不为“none”的元素，
  • perspective值不为“none”的元素

元素的显式和隐藏

  • 父元素 visibility 为 hidden，子元素设置为 visible，是可以支持显式的（这一点可以用在占位符上）
  • visibility 是支持 transition 的
  • outline 是一个不占任何空间的属性，因此可以解决那些涉及滚动条的问题
