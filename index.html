react-hook 记录

Posted on 2019-10-10

常识

  • hook 只能使用在函数式组件的最外层，不可以被使用在任何分支或循环中，也不可以被使用在普通函数中（自定义 hook 除外）
  • 自定义 hook 的状态是独立的，每个使用的自定义 hook 都会有单独的 state
  • 与 setState 不同，hook 中的状态更新总是以覆盖的形式而不是合并的形式
  • 函数式组件中的每一个局部变量都会随着每次渲染重新定义
  • fiber 可以解决大组件导致的渲染缓慢问题，具体而言是异步渲染
  • 异步渲染分为两个阶段 —— render parse 和 commit parse，前者包括一次 渲染的 redner 函数及之前全部步骤，完全可能被中断，可能会被反复执行多次，因此其中不应该有任何有副作用的东西。而后者只会执行一次，不会被中断，有副作用的请求可以放入其中。
  • suspense 的作用是进行代码分割，动态加载。 fallback 即为兜底逻辑，当加载页面时，发现所需组件尚在加载中，就会抛出一个 promise 错误，然后 suspense 会捕获该错误，当该 promise resolve 的时候，会进行重新渲染。

hook 与生命周期

  • useEffect Hook 看做 componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个函数的组合。
  • 每一次重新渲染都会生成新的 effect，替换掉之前的。effect 更像是渲染的结果，每个 effect 都属于一次特定的渲染。
  • effect 不会阻塞浏览器更新屏幕，大多数情况下 effect 也不需要同步执行。如果是一些特殊情况，可以使用 useLayoutEffect hook 来解决。
  • effect 在每一次渲染的时候都会执行，因此 react 会在执行当前 effect 前对上一个 effect 进行清除
  • 当使用 effect 的优化形式时，请确保数组中包含了所有外部作用域中会随时间变化并且在 effect 中使用的变量（包括间接使用，比如 effect 中调用的函数），否则你的代码会引用到先前渲染中的旧变量。
  • 如果你传入了一个空数组（[]），effect 内部的 props 和 state 就会一直拥有其初始值
  • 要记住 effect 外部的函数使用了哪些 props 和 state 很难。这也是为什么 通常你会想要在 effect 内部 去声明它所需要的函数。 这样就能容易的看出那个 effect 依赖了组件作用域中的哪些值
  • 在最坏的情况下，可以将函数放入 hook 的条件中，然后将函数的定义包裹进 useCallback 中
  • 当 effect 的依赖频繁变化时，可以考虑在 setState 使用回调函数的形式来解决问题(这种情况同样也是 useReducer 的使用场景)

其它 hook

  • useContext 会在离组件上层最近的 Context.Provider 更新时触发冲渲染，并使用最新的 context value
  • 在使用 useReducer 时，应当向子组件传递 dispatch 而不是回调函数
  • 要注意, useMemo 函数在渲染期间执行，请不要在这个函数内部执行与渲染无关的操作，注入副作用之类的操作属于 useEffect。
  • 当 ref 对象内容发生变化时，useRef 并不会通知你。变更 .current 属性不会引发组件重新渲染。如果想要在 React 绑定或解绑 DOM 节点的 ref 时运行某些代码，则需要使用回调 ref 来实现。

原理

  • react 通过 hook 的顺序来确认 state 和 useState 的对应关系
  • react 保证 useState 的函数是稳定的，并不会在组件重新渲染时发生新变化
  • react 通过 Object.is 来并判断是否更新，并以此为依据来处理渲染
  • 可以理解为每一次渲染都是完全新的组件，里面的 state 和 props 都是不变的（useRef 属于另一种情况）

css 世界

Posted on 2019-10-01

基础流布局简介

  • 流实际上是 css 世界中的一种基本定位和布局机制，具备自适应的特性。流对于某些 table 并不适用。
  • 通常把元素分为块级元素和内联元素
  • width 为 auto 时如果是块级元素会充分利用可用空间，如果是浮动、绝对定位、inline-block 时会收缩到合适大小，如果是内容连续的很长的英文或数字，则会超出（word-break:break-all 可以用来解决该问题）
  • 如果内部没有块级元素或块级元素没有设定宽度值，则最大宽度实际上是最大的连续内联盒子的宽度
  • 对普通流里的元素，如果 height:100% 要想生效，其父级必须有一个可以生效的高度值（否则百分比和 auto 无法计算）
  • width 不存在以上问题，它会使用包含块的真实的计算值作为百分比计算的基数
  • 正常布局的宽高百分比是以包含块的 content-box 来计算的，而绝对定位的宽高百分比是相对于 padding-box。
  • max-width 的优先级要高过 width，而 min-width 的优先级要高过 max-width
  • 通过设置 max-height 可以完成高度展开的动画
  • 每个行框的开始都会有宽度为 0 高度为 line-height 或 font-size 的内联块，用于撑起基本的高度(可以通过设置一个 inline-block 的空 span 来体现)

盒尺寸四大家族

content

  • 所有替换元素都是内联元素，其内容不受页面 css 上的影响，有着自身的尺寸
  • 若替换元素的固有尺寸有这个宽高比例，那么设置一个就会自动形成另一个，将图片设置为 block 不会影响其尺寸规则
  • 替换元素和非替换元素只隔了一个 content 属性，但是替换元素的固有尺寸是无法设置的
  • content 只能改变视觉呈现，而当右键保存时，还是原来的内容
  • content 生成的文本无法被选中，此外，还不能匹配 empty 类

    padding

  • 内联元素的竖直方向的 padding 是影响布局的（具体体现在会出现滚动条）
  • 对于非替换元素的内联元素，不仅 padding 不会加入行盒高度的计算，margin 和 border 也是如此，都是不计算高度，但实际上在内联盒周围发生了渲染
  • 内联元素的高度完全是受 font-size 的限制
  • 很多表单元素都内置了 padding

    margin

  • 只有当元素处于充分利用可用空间状态的时候，margin 才可以改变元素水平或竖直的可见尺寸
  • chrome 的滚动条为子元素大小超过 content-box 时出现，而 firfox 和 ie 是超过 padding-box 时出现，因此，子元素只能使用 margin-bottom 留白，而不是 padding-bottom
  • margin 和 padding 的百分比无论水平还是竖直都是按照块度来计算的
  • 在没有设置宽高还会自动填充的情况下，margin 如果一侧为定值，一侧为 auto，则 auto 占据剩余所有空间，若都为 auto，则平分距离
  • 当内部元素尺寸大时，说明可用空间都没有了，会当作 0 来处理
  • 内联替换元素垂直 margin 有效，不会发生 margin 合并的情况

    border

  • border-color 的默认值为 color 的值

内联元素与流

  • vertical-align: middle 并不是绝对的垂直居中对齐，只是一种近似效果
  • ex 可以用来保证与文字大小相一致
  • 行内元素的大小是 line-height 撑开的，line-height 保证了最小的高度，对于非替换元素的纯内联元素可以认为高度完全由 line-height 绝对
  • 内容区域高度受 font-family 和 font-size 双重影响，而 em-box 仅受 font-size 影响，通常内容区域高度要更高一些。
  • 内联替换元素和非替换元素放在一起时会共同形成一个行框盒子，line-height 决定其最小高度
  • css 中，计算行高的时候，一定不要向下舍入，而是要向上舍入
  • vertical-align 可以使用值来计算，负值向下，正值向上。
  • vertical-align 只能作用域内联元素和 display 为 table-cell 的元素上
  • 当为 table-cell 时，vertical-align 设置在自己身上，作用域子元素上
  • 把 font-size 和 line-height 一起设置可以解决间隙的问题
  • 非主动触发唯一的内联元素是不可能跑到计算元素之外的

流的破坏与保护

float

  • 浮动参考的元素是浮动元素对齐参考的实体，通常是行框盒子
  • clear 属性是让自身不能与前面的浮动元素相邻，根本不会对之后的浮动元素造成影响
  • 如果 clear:both 元素前面的元素就是 float，那么 margin-top 负值即使设置成 -9999 px 也不会有任何效果。

    BFC

  • display 为 inline-block、table-caption、table-cell
  • position 不为 static 和 relative 时
  • overflow 不为 visible 的块元素
  • contain 值为 layout、content或 paint 的元素
  • 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
  • 网格元素（display为 grid 或 inline-grid 元素的直接子元素）
  • 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
  • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

    overflow

  • 清除浮动最合适的不是 clear，而是 overflow，设置 为 hidden。如果要完成元素裁剪的同时在四周设置间隙，那么可以使用透明边框，此时内监局为 padding 时无能为力的。
  • 如果两个方向的 overflow 有一个是 visible 而另一个不是，那么 visible 会表现为 auto。
  • 要想实现 text-overflow: ellipsis,必须同时设置另外两个属性– white-space:nowrap,overflow:hidden。
  • 锚点定位（使用 id 进行跳转）会发生在普通的容器上，且由内向外（指如果存在多层会从最里面那层逐层向上）。
  • overflow:hidden 里的元素也是可以滚动的，可以通过锚点来进行跳转。

    absolute

  • 当 absolute 和 float 同时存在的时候，float 属性是无任何效果的。
  • absolute 使用的包含块是 padding-box 而非 content-box
  • 行内元素生成的包含块由前后内联盒子决定，和里面的细节毫无关系（比如说里面字体特别大撑开）
  • absolute 如果不设置 top 和 left 的话会处于原位置，只是会脱离文档流。

    absolute 和 overflow

  • 如果 overflow 不是定位元素，同时绝对定位元素和 overflow 容器之间也没有定位关系，则 overflow 无法对 absolute 元素进行裁剪，而如果 overflow 的属性值不是 hidden 而是 auto 或者是 scroll，即使绝对定位元素高宽比 overflow 元素高宽还要大，也不会出现滚动条。
  • absolute 被裁剪或 fixed 固定定位失效时，可能是 transform 的原因，

    absolute 其它

  • 当 absolute 只设置某个方向时，另一个方向的定位依旧保持相对定位特性。
  • clip 要起作用其元素必须是绝对定位或者固定定位

    relative

  • 相对定位的百分比值是给予包含块计算的
  • 当相对方向的属性值同时出现时，会取上左方向
  • 在使用 relative 时，力求范围较小
  • 使用普通元素替换 fixed 可以解决很多问题

层叠

  • 在同层叠关系下，后来居上，大者为上。

    基本层叠关系（自下而上）

  • 层叠上下文
  • 负 z-index
  • block 块状水平盒子
  • float 浮动盒子
  • inline 水平盒子
  • z-index 为 auto 或 0
  • 正 z-index

    层叠上下文

    层叠上下文的作用

  • 层叠上下文提供了一个基准，如果层叠上下文 A 在层叠上下文 B 之下，那么 A 上的元素绝对不可能位于 B 之上，无论 z-index 是多大
  • 层叠上下文内采取 z-index 谁大谁在上面的措施（其实没什么卵用，正常的文档流根本用不着 z-index，如果要使用必然已经触发的位移）
  • z-index 为负数时，依旧无法突破当前上下文。z-index 负值的渲染就是寻找第一个层叠上下文的过程，然后层叠顺序止步于这个层叠上下文元素。

    如何产生层叠上下文

    z-index 派（对于它们，产生的层叠上下文顺序为 z-index 的值）

  • 当 position 为 relative、absolute 且 z-index 不为 auto(其实 auto 时也产生了，可以认为 z-index 为 0)
  • z-index 不为 auto 的 flex 或 grid 子项（此时 z-index 必须为数值）
  • position 为 fixed 或 sticky

    其它（此时都认为 z-index 为 0）

  • opacity 属性值小于 1 的元素（参考 the specification for opacity），
  • transform 属性值不为 “none”的元素，
  • mix-blend-mode 属性值不为 “normal”的元素，
  • transform属性值不为“none”的元素
  • filter值不为“none”的元素，
  • perspective值不为“none”的元素

元素的显式和隐藏

  • 父元素 visibility 为 hidden，子元素设置为 visible，是可以支持显式的（这一点可以用在占位符上）
  • visibility 是支持 transition 的
  • outline 是一个不占任何空间的属性，因此可以解决那些涉及滚动条的问题

精通CSS笔记

Posted on 2019-09-21

语义化 CSS

  • 给元素添加类名时，即使类名明确用于样式，也不要以体现其视觉效果，正确的作法是用类型指出该组件的类型。比如说，将商品列表命名为 product-list 而不是 larged-centered-list
  • 在使用选择器时，不应当涉及太多的上下文
  • 要从结构中分离出皮肤，从内容中分离出容器，不要将字体、颜色和定位、浮动的样式写在一起
  • 要注意以下几点:
    • 以类名作为应用样式的主要手段
    • 类名要能顾名思义，清晰明了
    • 通过拆分出单一用途的规则来避免不必要的重复
    • 不要把元素类型和样式规则绑定到一起

CSS 选择器

子代和同辈选择符

  • a > b: 所有子代
  • a + b: 所有相邻同辈属性（位于 a 之后相邻的 b）
  • a ~ b: 所有同辈组合子，用于选择位于 a 之后的所有 b

属性选择器

  • a[title]: 拥有 title 属性的 a 标签
  • a[title=”submit”]: 拥有 title 属性为 submit 的 a 标签
  • a[href^=”abc”]: title 属性以 abc 开头
  • a[href$=”abc”]: title 属性以 abc 结尾
  • a[href*=”abc”]: title 属性中包含 abc 三个字符
  • a[href~=abc]: 经空格分隔开的值中有一个为 abc

伪类

  • target: 它匹配的元素拥有 id 属性，且为当前页面 URL 末尾的 # 后边的值
  • not(): 反选伪类
  • nth-child(3n + 4): 该元素的 第(n=0,1,2) 个元素
  • nth-child(-n + 4): 当 n 为负值时，即为前四个(nth-last-child 即为逆序)
  • only-of-type: 会选择特定类型的元素，如果为纯数字，则为总体而言的第 n 个（必然只出现一次）

层叠

  • 标注为 !important 的用户样式
  • 标注为 !important 的开发者样式
  • 开发者样式
  • 用户样式
  • 浏览器默认样式

字体

  • em 对字体是百分比叠加的（如果是用在 font-size 上是以父元素的 font-zize 为基准）
  • 行高指的是行盒子的总高度，更通俗的是行间距，排版属于叫铅空，就是排字用来隔开字符的铅块。css 中的铅空会始终同时应用于行盒子的上方和下方。
  • line-height 设置数字意味着以当前 font-size 为基准，子元素继承的会是一个系数，始终以自身 font-size 为基准
  • vertical-align 的默认值是 baseline，即子元素的基线与父元素的对齐
  • font-variant 可以通过 small-caps 把英文文本转换成所谓的小型大写字母，即全部字母都大写，但是只有第一个是正常大小，其它的字母的大小会和小写时一样
  • 使用新 css 属性 hyphens 可以在连续英文中插入连字符
  • column 模块可以将网页中部分的内容转换成类似与报纸的分栏效果，而 colimn-span 可以营造出横跨多栏的效果
  • 在排版时常见的是以正文行高的整数倍来进行设计，因此，margin line-height 的和应该也是整数倍
  • 潜入 web 字体的关键是 @font-face 规则，其中必须要有的是 font-family 和 src 属性，而 font-weight 和 font-style 则是可选的，后两者不会改变 web 字体，但是会影响匹配的优先级——浏览器会优先匹配正确的字体，正确的 font-family 会优先于正确的粗细值。
  • web Font Loader 可以报纸跨浏览器的字体加载的一致性（统一 FOCUS），匹配后备字体可以进行微调来减少抖动
  • 文字阴影通常用于标题或短文吧，需要指定 x，y 的偏移量以及模糊距离和颜色值，多组阴影会进行这边，先定义的在下，后定义的在上。

可见格式化模型

盒子模型

  • 组成盒子模型的范围有 width, height, padding, border, margin, outline 类似于 border，但是并不影响盒子和宽度和高度，因此可以用来复杂布局的调试。
  • 默认情况下，width 和 height 都指的是内容区的宽度和高度，但这样其实蛮坑爹的
  • 通过将 box-sizing 的值修改为 border-box，那么 width 和 height 属性的值将包含内边距和边框，不过 margin 不会算在其中
  • 在 CSS 中设置任何高度值都应该慎重，这是因为元素的高度通常应该取决于所包含的内容，一个定值容易产生越界。如果需要使用，也最好使用 min-height。

可见盒子模型

  • 行内盒子是沿文本流水平排列的，也会随着文本换行而换行。它们之间的水平间距可以通过水平方向的 margin、padding、border 来调节。而在竖直方向，唯一修改盒子大小的方法只有 line-height（line-height代表最小值）。经实验，感觉 line-height 影响的更像是行内元素的排放位置，并不影响相应区域
  • 当文本没有被任何元素包裹的时候，即为匿名盒子，所有展示都位于盒子中
  • 外边距折叠：外边距折叠只发生在文档常规文本流的垂直方向上，行内盒子、浮动盒子和绝对定位盒子的外边距不会折叠。当发生外边距折叠时，最终结果取较大的那个值，当出嵌套时，如果没有 border 或 padding 或者某些具体的内容在进行隔离，同样会出现外边距折叠，一个空的块级元素自身的上下边距也会折叠。在外边距折叠后，如果碰到其它外边距，还会继续折叠。
  • 包含块是计算内边距和外边距百分比的依据，在 static 中，默认使用父元素作为包含块。

浮动

  • 浮动元素会自动收缩为适应其中内容的最小宽度，除非显式设置的 width
  • 如果浮动元素太窄，无法容纳所有浮动元素水平排列，那么就会向下移动。
  • 由于浮动元素脱离正常的文档流中，因此无法在文档流内部生成高度，因此需要在结束的的 div 标签签名额外添加一个空元素，然后清除该元素。此外，还可以通过 ::after 伪类生成 content 来处理问题。

    格式化上下文（BFC）

  • 页面必须包含突出的浮动元素，而且所有块级盒子左边界都默认与包含块左边界对齐。这组规则即为块级格式化上下文。外边距折叠只会发生在属于同一块级格式化上下文的块级元素中。
  • 以下方法为生成块级格式化上下文的方法：
    • display 为 inline-block 或 table-ceil之类的元素
    • float 属性值不为 none 的元素
    • 绝对定位的元素
    • overflow 不为 visible 的元素
  • 浮动元素从文档流中脱离后，会触发其后的元素中行盒子的缩短行为，而当一个元素具备了触发新跨级格式化上下文的条件，且挨着浮动元素后，就会收缩至适当大小且保证做外边必为直线。

水平布局

层叠

  • 要注意，z-index 只有在父级层叠上下文中才有意义，子级层叠上下文会被视为单独的一部分。

    水平布局

  • 要让两个元素的内容竖直方向对齐，先把它们设置为 inline 属性，然后应用 vertical-align: center
  • 在 HTML 的行内，可能出现将换行视为一个空格的情况，这组情况下需要将 font-size 设置为 0 来解决
  • 通常只将行内块当作兼容旧版本浏览器的逻辑，正常而言应当使用更为先进的 flexbox
  • 表格布局与常规块不同，不设置宽度的表格会“收缩适应”内容宽度，除非包含块中的内容将其撑开，但是表格布局有很多弊端，比如无法应用外边距，再比如给其定位时可能产生一些无法预料的问题

    Flexbox

  • 在 flex 中，若不指定大小，那么 flex 容器内的项目会自动收缩，也就是说，会变成容纳各项的最小宽度（如果主轴为列那就是高度），以恰好能容纳自身内容为限。
  • 如果某一侧外边距值为 auto，且在容器那一侧还有空间，那么该外边距就会扩展至占据所有位置。
  • align-items 的默认值为 stretch，也就是说，会拉伸至充满父容器
  • 其实 flex + margin:auto 就可以完成单个元素的居中
  • flex-basis 的默认值为 auto，即为取默认的宽或高，如果没有设置，则收缩为可容纳的最小宽度，当设置为 0 时，flex-grow 将不再考虑其宽度所带来的影响
  • flex-grow 的默认值为 0，即默认不再扩展
  • flex-shrink 即为收缩，考虑时将使用 flex-basis * flex-shrink 作为比例系数来进行缩放
  • flexbox 的方向是逻辑方向，即以文本方向决定哪里是开头哪里是结尾
  • order 可以在不改变 html 的情况下改变展示顺序，order 越小越靠前，相等则按照原顺序
  • 为了处理 flex-box 换行后过于庞大的情况，可以给 item 设置最大宽度
  • 与常规块不同，不需要将可伸缩项设置为非 static 的定位值，也可以对它们应用 z-index 属性，并产生 z-index 上下文
  • flex 中可伸缩项具有隐性最小宽度，即收缩不可能收缩到无法展示内容的情况下，为此，可以覆盖 min-width，明确一个主尺寸
  • flex 第一个参数为 flex-grow，第二个为 flex-shink， 第三个是 flex-basis

盒子模型

  • 颜色的 hsl 语法需要传入三个参数，第一个是角度，第二个和第三个分别代表饱和度和亮度
  • opacity 设置之后，是无法再让其子元素变得不那么透明的

    背景图片

  • 在使用背景图片设置位置时，设置具体像素值代表着包含块左上角具体图片左上角的距离，而百分比则代表着包含块左上角到图片对应百分比点的距离正好为包含块的百分比。
  • 背景图片默认展现在 border 范围内，而 background-clip 属性可以改变这个行为，这个属性的默认值为 border-box，而 padding-box 和 content-box 可以解决这个问题。
  • background-attachment 可以解决背景图片附着的问题
  • background-size 是百分比并不是根据图片固有大小，而是根据包含块的大小来测定的
  • background-size 的 contain 和 cover 都不改变 宽高比例，前者可以保证完全被展示，后者会导致被剪切
  • backgound-posiiton 的百分比是特殊的，百分比值的偏移指定图片的相对位置和容器的相对位置重合。值0%代表图片的左边界（或上边界）和容器的左边界（上边界）重合。值100%代表图片的右边界（或下边界）和容器的右边界（或下边界）重合。值50%则代表图片的中点和容器的中点重合。

border

  • border 的顺序是左上角开始顺时针旋转
  • border 的值指的是圆的半径，若两个圆角的弧线相交，则两个轴向都会分别缩小半径，知道圆弧不再相交。对于方形元素的对称圆角而言，任何大于 50% 的值都会得到圆形。而对于矩形元素，则可以设置一个非常大的值来实现胶囊效果。
  • 当矩形变为圆角后，元素对布局的影响和之前是一样的，但是响应点击的范围是以变化之后为标准的。

    盒阴影

  • 头两个值表示 x 轴和 y 轴的偏移，第三个值表述模糊半径，最后一个值是颜色。
  • box-shadow 可以使用 inset 来设置内阴影，从而作出一种被镂空的效果

    渐变

  • 渐变必然是从一个方向到齐对称方向，默认是从上到下，坐标轴垂直向上，方向为顺时针。

    嵌入图片的改变

  • max-width 100% 是限制图片的方法，这意味着会随着容器的缩小而变小，但不会大到超过自身尺寸，且不会改变其比例。
  • 通常会加上 width、height 为 auto 的声明
  • object-fit 可以像 background-size 一样来设置图片的大小，默认是 fill，会拉伸图片来充满容器，因此可能破坏宽高比。none 代表使用图片自有宽高，cover 和 contain 与 background-size 相同，scale-down 则会从 none 和 contain 中选一个，以小的为准
  • 没有设置宽高的可替代元素最终的默认大小可能是 300px 或 150px
  • 内外边距的百分比是以本身的宽度来确定的，因此可以设置为宽度 100%，padding-bottom 75% 来完成视频的包含

页面布局与网格

  • 传统行列布局的方法是通过 row 和 col 的百分比来计算宽度，通过 padding 来完成边距自适应
  • flex 整体而言是更好的旋转，但是 flex box 会随着其中内容的加载而重新计算尺寸，因此在页面首次打开的时候会调一下，体验并不好

    二维网格布局

  • 分别通过 grid-template-rows 和 grid-template-columns 来设置宽高，1 fr 代表可分配宽度分之一，这个会放到最后来计算
  • minmax 可以设置以下最小值
  • 当 grid-row 或 grid-column 设置跨所有行时，可以将最后一个置为 -1
  • justify-content 和 align-content 可以设置多个个体的排布
  • 而 align-self 可以设置单个在副轴上的排布
  • grid-auto-flow 可以设置默认主轴的方向
  • 默认排布模式是会导致空洞的（出现较大的元素），如果设置为稠密模式（grid-auto-flow: row dense）会自动跑多变，每次都从头开始，尽力找到最前面得到空位置
  • 可以通过 grid-area 来给网格空间命名

响应式 web 设计与 css

  • 理想视口即手机应该有的大小，而可见时刻为实际显式的内容，布局视口则是放大后的整个页面的内容
  • 设置 meta 标签可以很好的使用理想尺寸，此外，css理论上也可以做到，在 style 标签中使用 @viewport 即可，最好放在 html 里，以保证被尽早执行
  • 响应式 web 的核心在于媒体查询，媒体查询可以使用一连串的逗号来解决，逗号相当于 或
  • 媒体查询中使用的 em 始终相对于浏览器偏好中的基准字段，而不是可以通过 css 调整的 html 元素的 rem
  • 响应式设计应当优先于移动端，flexbox 和 grid 其实都是很不错的旋转
  • 通过 img 的 srcset 属性，可以让浏览器对图片自有进行预处理，从而让图片等资源在完成整个页面晕眩之前就进行加载
  • srcset 可以针对不同的大小和分辨度来使用不同的图片
  • sizes 属性可以针对当前的图片来决定图片的大小
  • 可以在 picture 属性中使用 webp，其中有 source 和 img，img 可以作为整体的后备

变换、过度与动画

变换

  • transform 不会影响布局本身，但是所有点都会被移到新位置上，图形所在的整个局部坐标系统都会被旋转，同样，旋转也不会影响页面其它部分的布局
  • 变换默认是以元素边框盒子的中心为原点的，可以通过 transform-origin 来设置 1 到 3 个值，如果忽略，则为 center。坐标原点在图形左上方。
  • svg 的变换有一些不同，比如 transform-origin 默认即为左上角，而非中心。
  • translate 的百分比是相较于自身而非包含块
  • 多重变换的顺序非常重要。如果先旋转伪元素，那么变换会以旋转后的坐标来完成
  • 应用给元素的任何变换都会创建一个堆叠上下文，这意味着在使用变换时，要注意 z-index，因为变换后的元素有自己的堆叠上下文，也就是说，子元素上的 z-index值再大，也无法和父堆叠上下文呢相比较。
  • 在进行 x，y 轴的旋转时，会涉及到透视的概念，用户距离透视点越近，影响越大，反之越小，默认无穷远，即不影响。
  • 默认情况下，嘉定观察者的实现与应用透视的元素相较于元素的中心。通过 perspective-origin 属性来修改消失的点的位置。
  • backface-visibility 属性可以用来控制背面展示，默认是 visible，不过可以设置为 hidden
  • 当进行三维反转时，所有子元素的三维反转都会失效，因此需要通过 transform-style 属性来设置一个三维上下文， 改制为 perserve-3d.

    transition 动画

  • 单写某个 transition 形式无法制定多个属性，但可以指定多个不同的过度，以逗号进行分隔。
  • transition-property 指定的值决定了要应用的过度数量，如果其它过度列表持续时间更短，则会重复。
  • steps 可以将动画分为不同的几帧。
  • transition-delay 可以用来延迟动画的触发，延迟时间可以为负值，这样可以跳过一部分动画。
  • 直接修改 dom 的 css 属性可以触发 dom

缺陷

  • transition需要事件触发，所以没法在网页加载时自动发生。
  • transition是一次性的，不能重复发生，除非一再触发。
  • transition只能定义开始状态和结束状态，不能定义中间状态，也就是说只有两个状态。
  • 一条transition规则，只能定义一个属性的变化，不能涉及多个属性。

    关键帧动画

  • 关键帧动画中 css 属性不允许添加 ！important，加了会被忽略。
  • animation-fill-mode 可以用来处理动画结束后的状态，比如 none 为没开始，backwards 为第一针，forwards 为结束状态，both 则轮流应用 forards 和 backwards
  • animation-play-state 用来处理动画播放一半时中断的状态
  • animation-direction 主要是 normal 和 reverse

高级特效

  • shape-outside 可以应用给浮动元素，不改变自身的外观，而是通过设置其形状影响外部的文档流
  • 当 shape-outside 设置为 border-box 时，此时设置 margin 会呈现 border-box 的形状
  • 剪切会影响对象的相应区域，而蒙版不会
  • 剪切使用 clip-path 来进行，只影响文档渲染后的外观和相应区域，并不会影响文档流。
  • 剪切可以通过 svg，具体而言是在 svg 里面设置 id，在引用时使用 url + id 的形式来解决。
  • 由于 svg 下剪切路径硬编码的坐标，因此无法缩放，可以将其设置为大小为 1 的 svg 来解决这个问题（换成了对象边界坐标系）
  • 为保证正确的几个，我们要遵循应用剪切、蒙版、混合和滤镜的标准次序

原理

浏览器如何解析 css

  • 解析文件即构建对象模型（将 html 解析为 dom 对象节点树，在浏览器碰到指向 css 的链接时，会获取并解析 css，将 css 文件转化为 css 对象模型，同样是树型结构）
  • 构建渲染树（被隐藏的 DOM 节点不会出现在这里，而类似于 :before 的伪元素则可能出现在其中，但是在 DOM 中不会有，除了节点，浏览器还会在这里保存其它表现性信息，如滚动块或视口）
  • 布局（根据渲染树确定不同对象显式于页面的什么位置）
  • 给予渲染树的信息进行绘制（如果浏览器发现最终图像的某一部分不影响页面其他内容的显式，也可能会拆分成不同的任务，每个任务负责绘制特定的部分）
  • https://csstriggers.com/ 标注了每个属性对应渲染的什么阶段
  • will-change 属性可以用来进行硬件加速，但是一定要慎用

性能

  • 尽量减少 HTTP 请求，将 css 控制在 1～2 个范围内，注意，@import 同样会触发 HTTP 请求
  • 压缩和缓存可以有效的加快加载速度
  • script 会阻塞渲染，而 async 会异步加载脚本，但是会在脚本加载完毕时执行，阻断 HTML 解析。如果时 defer 的话，同样会异步加载脚本，但是会等到 HTML 解析结束后再执行加载的脚本

杂项

  • 如果希望根据浏览器是否支持某个 CSS 属性来提供不同的样式，那么可以在 css 中使用 @supports 模块，该代码被成为条件规则

基本介绍

背景

Thrift 是一种框架，可以创建高效的、无缝的服务，其传输数据采用二进制格式，相对 XML 和 JSON 体积更小，对于高并发、大数据量和多语言的环境更有优势。此外，作为一种 RPC 框架，Thrift 可以让人几乎无感知的进行调用。

数据结构

Thrift 脚本可定义的数据类型包括以下几种类型：

基本类型：

  • bool：布尔值，true 或 false，对应 Java 的 boolean
  • byte：8 位有符号整数，对应 Java 的 byte
  • i16：16 位有符号整数，对应 Java 的 short
  • i32：32 位有符号整数，对应 Java 的 int
  • i64：64 位有符号整数，对应 Java 的 long
  • double：64 位浮点数，对应 Java 的 double
  • string：未知编码文本或二进制字符串，对应 Java 的 String

    结构体类型：

  • struct：定义公共的对象，类似于 C 语言中的结构体定义（可以嵌套）

    容器类型：

  • list：对应 Java 的 ArrayList
  • set：对应 Java 的 HashSet
  • map：对应 Java 的 HashMap
  • enum：

    异常类型：

  • exception：对应 Java 的 Exception

    服务类型：

  • service：对应服务的类, 其实更接近一个接口，里面对于方法的入参和返回值进行了定义

配置语法

  • include: 将某个 thrift 包括在内
  • typedef: typedef i32 MyInteger,定义自己的类型
  • const: 定义常量
  • optional: 在 struct 里可以定义可选项

使用介绍

分为 client 端和 server 端，client 端引入生成的 client 后就可以使用了，而 server 端需要将 thrift 文件里定义的方法都进行一次实现才可以使用。

  • 初始化: go mod init github.com/you/hello
  • 获取新依赖包: go get
  • 查看所有依赖包的版本: go list -m all
  • 更新所有直接和间接依赖到最新版本: go get -u
